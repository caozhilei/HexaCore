# 出入口（Entry Points）层技术方案文档

## 1. 概述

### 1.1 定位与职责

出入口层是六元组架构中的统一协议转换层，负责将所有异构输入标准化为统一的内部事件对象。基于HexaCore框架的Channel插件架构实现，出入口层的核心职责包括：

- **多渠道协议适配**：支持WhatsApp、企业微信、Web等多渠道接入，消除平台差异
- **输入标准化**：将所有渠道输入转换为HexaCore标准的InboundMessage对象
- **安全策略执行**：实现渠道级精细化安全控制，支持三种DM策略
- **性能优化**：通过连接池管理、消息批处理等机制确保高并发场景下的性能
- **热插拔扩展**：支持新增渠道适配器插件，无需修改核心代码

### 1.2 在六元组架构中的位置

出入口层作为六元组架构的输入边界，承担着"协议转换桥"的角色：

```
用户请求 → 出入口层 → 标准化InboundMessage → 路由层 → 频道层 → ...
```

与六元组其他元组的协作关系：
- **与路由层协作**：提供标准化消息对象，支持7级匹配规则
- **与频道层协作**：传递Session Key信息，确保上下文隔离
- **与技能层协作**：提供原始请求上下文，支持技能调用
- **与记忆层协作**：提供消息元数据，支持会话历史管理
- **与沙箱层协作**：传递安全策略，确保输入验证和隔离

### 1.3 设计目标

基于HexaCore框架的设计目标：

1. **HexaCore兼容性**：完全遵循HexaCore Channel接口标准，确保与HexaCore Gateway无缝集成
2. **企业级安全**：与HexaCore安全模型对齐，实现白名单、配对、开放三种DM策略
3. **高性能**：支持异步消息处理，确保平均响应时间<5秒的性能目标
4. **可扩展性**：支持插件化架构，新增渠道可在1小时内完成适配
5. **成本优化**：通过消息压缩和批处理，降低网络开销和资源消耗

### 2.2 协议转换流程

完整的协议转换流程遵循HexaCore事件处理模型：

1. **原始消息接收**：各渠道适配器接收平台原始消息（Webhook、WebSocket、HTTP等）
2. **协议解析**：解析渠道特定协议格式，提取关键信息
3. **标准化转换**：转换为HexaCore标准的InboundMessage对象
4. **安全验证**：应用安全策略，验证消息合法性
5. **事件发布**：将标准化消息发布到Gateway事件总线
6. **路由分发**：由路由层进行智能分发到对应Agent

### 2.3 HexaCore集成点

出入口层与HexaCore框架的核心集成点：

1. **Channel接口实现**：遵循HexaCore的`Channel`接口定义，实现`start`、`stop`、`send`等方法
2. **配置管理**：使用HexaCore标准的`HexaCore.json`配置文件格式（JSON5）
3. **安全模型**：集成HexaCore的DM策略系统，支持allowlist、pairing、open三种模式
4. **事件系统**：发布HexaCore标准格式的`inbound_message`事件
5. **遥测集成**：与HexaCore遥测系统集成，提供渠道级监控指标

```typescript
      emitOwnEvents: true,
```

## 5. 性能优化策略

### 5.1 连接池管理

为每个渠道实现智能连接池，优化资源利用：

```typescript
// src/entry-points/channels/shared/connection-pool.ts
export class ConnectionPool {
  private pool: Map<string, Connection>;
  private idleConnections: Set<string>;
  private metrics: PoolMetrics;
  
  constructor(
    private maxSize: number = 100,
    private idleTimeout: number = 300000, // 5分钟
    private healthCheckInterval: number = 60000, // 1分钟
  ) {
    this.pool = new Map();
    this.idleConnections = new Set();
    this.metrics = new PoolMetrics();
    
    // 启动健康检查定时器
    setInterval(() => this.healthCheck(), healthCheckInterval);
  }
  
  async acquire(connectionId: string): Promise<Connection> {
    // 1. 检查现有连接
    let connection = this.pool.get(connectionId);
    
    // 2. 如果不存在或不可用，创建新连接
    if (!connection || !connection.isHealthy()) {
      connection = await this.createConnection(connectionId);
      this.pool.set(connectionId, connection);
      this.metrics.connectionCreated();
    }
    
    // 3. 从空闲集合移除
    this.idleConnections.delete(connectionId);
    
    // 4. 检查池大小，必要时清理最旧的空闲连接
    if (this.pool.size > this.maxSize) {
      await this.evictOldestIdle();
    }
    
    return connection;
  }
  
  async release(connectionId: string): Promise<void> {
    const connection = this.pool.get(connectionId);
    if (connection) {
      this.idleConnections.add(connectionId);
      this.metrics.connectionReleased();
      
      // 设置空闲超时
      setTimeout(() => {
        if (this.idleConnections.has(connectionId)) {
          this.closeConnection(connectionId);
        }
      }, this.idleTimeout);
    }
  }
  
  private async evictOldestIdle(): Promise<void> {
    const oldest = Array.from(this.idleConnections).slice(0, this.idleConnections.size - this.maxSize);
    for (const id of oldest) {
      await this.closeConnection(id);
    }
  }
}
```

### 5.2 消息批处理

实现智能批处理机制，减少网络开销：

```typescript
// src/entry-points/channels/shared/message-batcher.ts
export class MessageBatcher {
  private batchQueue: Array<InboundMessage>;
  private batchTimer: NodeJS.Timeout | null;
  private processing: boolean;
  
  constructor(
    private batchSize: number = 50,
    private batchInterval: number = 100, // 100ms
    private processor: BatchProcessor,
  ) {
    this.batchQueue = [];
    this.batchTimer = null;
    this.processing = false;
  }
  
  async addMessage(message: InboundMessage): Promise<void> {
    this.batchQueue.push(message);
    
    // 触发条件：达到批量大小或定时器触发
    if (this.batchQueue.length >= this.batchSize) {
      await this.processBatch();
    } else if (!this.batchTimer) {
      this.batchTimer = setTimeout(() => this.processBatch(), this.batchInterval);
    }
  }
  
  private async processBatch(): Promise<void> {
    if (this.processing || this.batchQueue.length === 0) {
      return;
    }
    
    this.processing = true;
    
    try {
      // 清理定时器
      if (this.batchTimer) {
        clearTimeout(this.batchTimer);
        this.batchTimer = null;
      }
      
      // 提取当前批次
      const batch = this.batchQueue.splice(0, this.batchSize);
      
      // 处理批次
      await this.processor.process(batch);
      
      // 如果还有剩余消息，继续处理
      if (this.batchQueue.length > 0) {
        setImmediate(() => this.processBatch());
      }
    } finally {
      this.processing = false;
    }
  }
}
```

### 5.3 错误重试机制

实现基于指数退避的智能重试策略：

```typescript
// src/entry-points/channels/shared/retry-manager.ts
export class RetryManager {
  async withRetry<T>(
    operation: () => Promise<T>,
    options: RetryOptions = {},
  ): Promise<T> {
    const {
      maxRetries = 3,
      baseDelay = 100,
      maxDelay = 10000,
      retryableErrors = ['NETWORK_ERROR', 'TIMEOUT'],
    } = options;
    
    let lastError: Error;
    
    for (let attempt = 0; attempt <= maxRetries; attempt++) {
      try {
        return await operation();
      } catch (error) {
        lastError = error as Error;
        
        // 检查是否可重试
        if (!this.isRetryable(error, retryableErrors) || attempt === maxRetries) {
          throw error;
        }
        
        // 计算延迟时间（指数退避）
        const delay = Math.min(
          baseDelay * Math.pow(2, attempt) * (0.5 + Math.random()),
          maxDelay,
        );
        
        // 等待重试
        await this.sleep(delay);
      }
    }
    
    throw lastError;
  }
  
  private isRetryable(error: any, retryableErrors: string[]): boolean {
    const errorCode = error.code || error.name;
    return retryableErrors.includes(errorCode);
  }
  
  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}
```

### 5.4 缓存优化

```typescript
// src/entry-points/channels/shared/cache-manager.ts
export class CacheManager {
  private memoryCache: LRUCache<string, any>;
  private redisClient: RedisClient;
  
  constructor(
    private memoryLimit: number = 1000,
    private redisConfig?: RedisConfig,
  ) {
    this.memoryCache = new LRUCache({
      max: memoryLimit,
      ttl: 60000, // 1分钟
    });
    
    if (redisConfig) {
      this.redisClient = new RedisClient(redisConfig);
    }
  }
  
  async get<T>(key: string): Promise<T | null> {
    // 1. 检查内存缓存
    const memoryValue = this.memoryCache.get(key);
    if (memoryValue !== undefined) {
      return memoryValue as T;
    }
    
    // 2. 检查Redis缓存
    if (this.redisClient) {
      const redisValue = await this.redisClient.get(key);
      if (redisValue) {
        // 回填到内存缓存
        this.memoryCache.set(key, redisValue);
        return JSON.parse(redisValue) as T;
      }
    }
    
    return null;
  }
  
  async set(key: string, value: any, ttl?: number): Promise<void> {
    // 1. 设置内存缓存
    this.memoryCache.set(key, value);
    
    // 2. 设置Redis缓存
    if (this.redisClient) {
      await this.redisClient.set(
        key,
        JSON.stringify(value),
        ttl ? { EX: ttl } : undefined,
      );
    }
  }
}
```

```typescript
// src/entry-points/shared/dm-policy.ts
export class DMPolicyManager {
  async checkPolicy(message: InboundMessage, config: HexaCoreConfig): Promise<PolicyResult> {
    const channelConfig = config.channels[message.channel];
    const policy = channelConfig?.dmPolicy || 'open';
    
    switch (policy) {
      case 'allowlist':
        return await this.checkAllowlist(message, channelConfig.allowlist || []);
        
      case 'pairing':
        return await this.checkPairing(message, channelConfig.pairingConfig);
        
      case 'open':
      default:
        return { allowed: true, reason: 'open_policy' };
    }
  }
  
  private async checkAllowlist(message: InboundMessage, allowlist: string[]): Promise<PolicyResult> {
    const isAllowed = allowlist.includes(message.peer.id);
    return {
      allowed: isAllowed,
      reason: isAllowed ? 'in_allowlist' : 'not_in_allowlist',
      details: {
        peerId: message.peer.id,
        allowlistSize: allowlist.length,
      },
    };
  }
  
  private async checkPairing(message: InboundMessage, config: PairingConfig): Promise<PolicyResult> {
    const pairingStatus = await this.getPairingStatus(message.peer.id);
    const isPaired = pairingStatus === 'paired';
    
    return {
      allowed: isPaired,
      reason: isPaired ? 'already_paired' : 'not_paired',
      details: {
        peerId: message.peer.id,
        pairingStatus,
        pairingRequired: config.requirePairing,
      },
    };
  }
}
```
