# HexaCore 技术方案文档 (v3)

## 1. 架构总览

HexaCore 采用微服务与模块化相结合的架构，后端核心服务通过 Docker 编排，前端管理后台基于 Next.js 构建。

### 1.1 系统架构图
```
[User] <--> [Kong Gateway :8000] <--> [Supabase Services]
                  ^                          ^
                  |                          |
[Web Dashboard] --+                          |
(:3000)                                      |
                                             v
                                     [HexaCore Backend]
                                     (Node.js/TS)
                                     - Orchestrator
                                     - Routing Engine
                                     - LLM Service
                                     - Skill Manager
```

## 2. 技术栈

### 2.1 后端核心
- **Runtime**: Node.js 18+
- **Language**: TypeScript 5.x
- **Framework**: 原生模块化设计 (无重型框架依赖)
- **Database**: Supabase (PostgreSQL 15+)
- **ORM**: Supabase JS Client (RESTful over Postgres)

### 2.2 前端 (Web Dashboard)
- **Framework**: Next.js 14 (App Router)
- **UI Library**: shadcn/ui + Tailwind CSS
- **State**: React Hooks + SWR (Supabase SSR)
- **Deployment**: Docker Standalone

### 2.3 LLM 集成
- **Provider**: 自研 `LLMService` 抽象层
- **Supported Models**:
  - OpenAI (GPT-3.5/4)
  - DeepSeek (V2/V3)
  - Qwen (Turbo/Plus/Max)
- **Protocol**: OpenAI-compatible API

## 3. 核心模块实现

### 3.1 路由引擎 (Routing Engine)
- **位置**: `src/core/routing/`
- **机制**:
  - 加载 `routing_rules` 表中的规则。
  - `CompositeMatcher` 遍历执行匹配 (Peer, Keyword, Semantic)。
  - 返回最佳匹配 `AgentID`。
- **扩展性**: 支持动态添加新的 Matcher 类。

### 3.2 编排器 (Orchestrator)
- **位置**: `src/core/orchestrator.ts`
- **职责**:
  1. 接收 `InboundMessage`。
  2. 调用 `RoutingEngine` 获取目标 Agent。
  3. 初始化/获取 Session。
  4. 加载 Memory 上下文。
  5. 构建 Prompt 并调用 `LLMService`。
  6. 保存 Memory 并返回结果。

### 3.3 数据库设计 (Supabase)
- **Agents**: 存储 Agent 配置、Prompt、模型参数。
- **Sessions**: 存储会话状态、当前步骤。
- **Memories**: 存储对话历史 (Text) 和向量索引 (Embedding)，支持 `session_key` 关联或 `space_id` 关联。
- **Memory_Spaces & Grants (v3.1新增)**: 支持记忆空间（会话私有或工作组共享）与 `read/write` 授权机制。
- **Routing_Rules**: 存储动态路由规则。

### 3.4 Web 管理后台
- **功能**:
  - **Auth**: 基于 Supabase Auth 的用户认证。
  - **Agent CRUD**: 可视化配置 Agent 属性。
  - **Chat**: WebSocket 实时对话组件 (`src/components/chat`).
- **权限**:
  - RLS 策略强制隔离用户数据。

## 4. 部署方案

### 4.1 Docker Compose
所有服务通过 `docker-compose.yml` 统一编排：
- `supabase-db`: PostgreSQL 数据库
- `supabase-auth`: GoTrue 认证服务
- `supabase-rest`: PostgREST API 服务
- `supabase-realtime`: WebSocket 实时服务
- `kong`: API 网关
- `web-dashboard`: Next.js 前端应用

### 4.2 环境变量
核心配置通过 `.env.local` 管理：
- `SUPABASE_URL`, `SUPABASE_ANON_KEY`
- `LLM_PROVIDER`, `LLM_MODEL`, `API_KEY`

## 5. 扩展性设计

### 5.1 技能 (Skill) 扩展
- 在 `src/core/skills/` 下实现 `SkillBase`。
- 在 Agent 配置中启用 Skill 名称。
- `Orchestrator` 根据配置自动注入 Skill 描述到 System Prompt (Function Calling)。

### 5.2 渠道 (Channel) 扩展
- 实现 `ChannelAdapter` 接口。
- 将外部协议转换为 `InboundMessage`。
- 注册到 `Orchestrator`。
