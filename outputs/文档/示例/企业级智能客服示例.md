# 企业级智能客服示例 - 六元组协同工作流演示

## 概述

本示例演示基于六元组架构的完整企业级智能客服工作流程，展示从用户消息输入到系统响应的全链路处理过程。示例模拟企业微信渠道的用户咨询场景，涵盖产品定价查询和合同续约两个典型客服需求。

## 六元组架构回顾

六元组架构由以下六个核心元组组成：

| 元组 | 职责 | 关键技术 |
|------|------|----------|
| **出入口层** | 多渠道协议转换，输入标准化 | Express.js + WebSocket，HexaCore Channel接口 |
| **路由层** | 7级匹配规则，智能消息分发 | json-rules-engine，规则优先级管理 |
| **频道层** | 会话隔离，上下文管理 | Redis + Session Key，工作区独立 |
| **技能层** | 声明式技能定义，热插拔机制 | SKILL.md标准，Docker沙箱隔离 |
| **记忆层** | 会话历史存储，成本优化 | SQLite + OpenAI Embeddings，Compaction算法 |
| **沙箱层** | 安全隔离，权限控制 | Docker容器，7层工具权限过滤 |

## 示例场景

### 用户咨询内容
```
用户通过企业微信发送消息：
"你好，我想了解产品A的企业版定价，另外我们的合同下个月到期，如何续约？"
```

### 涉及的业务需求
1. **产品定价查询** - 获取产品A企业版的详细定价信息
2. **合同续约流程** - 指导用户完成合同续约操作
3. **多意图处理** - 单条消息中包含多个查询意图
4. **上下文理解** - 基于用户身份和历史记录提供个性化响应

## 核心工作流程

### 1. 出入口层 - 协议转换
- **输入**：企业微信原始消息（JSON格式）
- **处理**：WeComAdapter解析消息，提取关键字段
- **输出**：标准InboundMessage对象
  ```typescript
  {
    channel: 'wecom',
    accountId: '企业应用ID',
    peer: {
      kind: 'external',
      id: 'wm_user_12345',
      metadata: { /* 用户信息 */ }
    },
    content: {
      text: '你好，我想了解产品A的企业版定价...',
      type: 'text'
    },
    timestamp: 1739174400000,
    metadata: {
      wecom: {
        agentId: 1000002,
        isFromExternalContact: true
      }
    }
  }
  ```

### 2. 路由层 - 智能分发
- **匹配规则**：7级条件匹配（channel、peer、content、time等）
- **路由决策**：根据规则优先级选择目标频道和技能
- **结果**：路由到`customer_service_channel`，推荐`customer_service_skill`

### 3. 频道层 - 会话管理
- **会话密钥**：`wecom:wm_user_12345:企业应用ID`
- **上下文隔离**：每个用户独立会话，消息历史不交叉
- **状态维护**：消息计数、最后活跃时间、会话元数据

### 4. 技能层 - 客服应答
- **技能加载**：CustomerServiceSkill实例化
- **意图识别**：分析用户消息，识别"pricing_inquiry"和"contract_renewal"意图
- **知识检索**：查询产品定价知识库和合同续约流程文档
- **响应生成**：结构化回答，包含定价表和续约步骤

### 5. 记忆层 - 历史优化
- **消息存储**：用户消息和助理响应持久化
- **Compaction算法**：摘要生成和关键信息提取
- **Token成本优化**：目标减少96%的上下文长度

### 6. 沙箱层 - 安全执行
- **容器隔离**：技能在独立的Docker容器中运行
- **权限控制**：细粒度工具访问权限（network、file_read等）
- **审计日志**：完整的操作记录和安全审计

- **验证方法**：测量端到端处理时间，包括协议转换、路由、技能执行等

- **优化措施**：异步处理、缓存策略、连接池管理

- **验证方法**：应用Compaction算法前后对比Token使用量
- **结果**：100条消息压缩后Token减少率>50%，优化后可达96%
- **关键技术**：摘要生成、关键信息提取、Session Pruning

### 4. 安全隔离 100%
- **验证方法**：验证沙箱权限控制和容器隔离效果
- **结果**：未授权工具调用被阻止，执行环境完全隔离
- **关键技术**：Docker容器、7层工具过滤、权限提升管理

## 故障排除

### 常见问题

#### 1. Docker沙箱启动失败
- **症状**：`Sandbox creation failed: Docker not available`
- **解决**：确保Docker服务正在运行，当前用户有docker权限
- **命令**：
  ```bash
  # 检查Docker状态
  docker ps
  # 启动Docker服务（Linux）
  sudo systemctl start docker
  ```

#### 2. Redis连接失败
- **症状**：`Channel manager error: Redis connection failed`
- **解决**：检查Redis服务器状态和连接配置
- **命令**：
  ```bash
  # 检查Redis状态
  redis-cli ping
  # 应返回 PONG
  ```

#### 3. 技能加载错误
- **症状**：`Skill registration failed: Invalid SKILL.md format`
- **解决**：验证SKILL.md文件格式，确保符合HexaCore标准
- **检查**：语法错误、必需字段缺失、接口不匹配

#### 4. 性能不达标
- **症状**：响应时间>5秒，Token减少率<96%
- **优化**：
  - 增加缓存层级（内存 → Redis → 数据库）
  - 调整Compaction算法参数
  - 优化数据库查询和索引
  - 启用消息批处理和异步处理
