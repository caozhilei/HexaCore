# HexaCore 智能体工作平台 需求分析文档 (v3)

## 1. 项目概述

### 1.1 项目简介
HexaCore 是一个基于**六元组架构 (6-Tuple Architecture)** 的企业级自主进化智能体工作平台。它旨在解决多渠道接入、智能路由、上下文隔离、技能扩展、记忆持久化和安全沙箱等核心挑战，为企业提供可控、可扩展的 AI 智能体解决方案。

### 1.2 核心价值
- **统一接入**: 屏蔽 WhatsApp、企业微信、Web 等渠道差异。
- **智能调度**: 基于规则和语义的精准路由分发。
- **安全隔离**: 会话级和 Agent 级的数据与权限隔离。
- **国产化支持**: 原生支持 DeepSeek (深度求索) 和 Qwen (通义千问) 等国产大模型。
- **可视管理**: 提供完整的 Web 管理后台，降低运维门槛。

## 2. 功能需求

### 2.1 出入口 (Entry Points)
- **多渠道适配**:
  - **CLI**: 命令行交互界面，用于开发调试。
  - **Web**: WebSocket 实时对话接口，支持前端集成。
  - **API**: RESTful API 接收外部 Webhook。
- **消息标准化**: 所有输入转化为统一的 `InboundMessage` 格式。

### 2.2 路由 (Routing)
- **多级匹配**:
  - Priority 0-100 优先级控制。
  - 支持 Peer ID、Metadata、Content Keywords、Semantic 等匹配条件。
- **数据库驱动**: 路由规则存储于 Supabase `routing_rules` 表，支持热更新。
- **默认策略**: 无匹配规则时路由至默认 Agent。

### 2.3 频道 (Channels) & 会话
- **会话隔离**: 基于 `SessionKey` (Agent:Channel:Peer) 实现上下文隔离。
- **状态管理**: 实时同步会话状态 (Supabase Realtime)。
- **持久化**: 会话历史记录存储于 PostgreSQL。

### 2.4 智能体 (Agents) & LLM
- **多模型支持**:
  - **OpenAI**: GPT-3.5/4
  - **DeepSeek**: deepseek-chat, deepseek-coder
  - **Qwen**: qwen-turbo, qwen-plus, qwen-max
- **Agent 配置**:
  - 自定义 System Prompt。
  - 模型参数调整 (Temperature, MaxTokens)。
- **记忆管理**:
  - 短期记忆: 上下文窗口内的最近对话（基于 `session_key` 隔离）。
  - 长期记忆: 基于 pgvector 的向量检索。
  - **共享记忆空间 (v3.1新增)**: 引入 `memory_spaces` 与基于权限的共享机制，允许工作组内的多个智能体跨会话共享黑板与团队知识库。

### 2.5 管理后台 (Web Dashboard)
- **认证**: 邮箱/密码登录 (Supabase Auth)。
- **Agent 管理**: 创建、编辑、删除 Agent。
- **调试工具**: 内嵌实时聊天窗口，验证 Agent 行为。
- **仪表盘**: 查看 Agent 列表和基础状态。

## 3. 非功能需求

### 3.1 性能
- **响应延迟**: CLI/Web 接口平均响应 < 3秒 (依赖 LLM 推理速度)。
- **并发能力**: 单容器支持 50+ 并发会话。

### 3.2 安全
- **鉴权**: 统一使用 Supabase Auth (JWT)。
- **数据权限**: RLS (Row Level Security) 确保用户只能访问自己的 Agent。
- **密钥管理**: API Key 通过环境变量注入，不硬编码。

### 3.3 可维护性
- **容器化**: 全栈 Docker 化部署 (API Gateway, DB, App, Web)。
- **配置化**: 核心参数通过 `.env` 配置。

## 4. 系统架构 (六元组实现)

| 元组 | HexaCore 实现 |
| :--- | :--- |
| **Entry Points** | `src/core/entry-points/` (WebAdapter, CLIAdapter) |
| **Routing** | `src/core/routing/` (RoutingEngine, Matchers, DatabaseRules) |
| **Channels** | `src/core/database/session-repo.ts` (Session Management) |
| **Skills** | `src/core/skills/` (SkillManager, BaseSkill) |
| **Memory** | `src/core/database/memory-repo.ts` (Supabase Storage) |
| **Sandbox** | Docker Isolation (部署层级) |
